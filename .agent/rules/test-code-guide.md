## 테스트 코드 작성 가이드

### 1. 파일 위치 및 환경

- **파일 위치**: 테스트 파일은 대상 파일과 **같은 경로**에 작성합니다.
  - 예시: `sum.ts` &rarr; `sum.spec.ts`
- **프레임워크**: `vitest`를 사용합니다.

### 2. 네이밍 및 스타일

- **메서드**: `it` 메서드를 사용합니다.
- **테스트 명**: `it` 함수의 설명(description)은 **한글**로 작성합니다.

### 3. 구조 및 데이터 관리

- **공통 데이터**: 반복해서 사용되는 데이터는 테스트 파일 **최상단**에 정의합니다.
- **공통 로직**: 반복되는 로직은 `beforeEach`, `beforeAll` 등의 훅(Hook) 메서드를 적극 활용합니다.

### 4. DOM 선택 전략

- **Selector**: 객체나 요소를 선택할 때는 가능한 한 `data-testid` 속성을 적용하여 선택합니다.

### 5. 테스트 실행 및 검증

- **실행**: 테스트 작성 후에는 `npx vitest run <파일경로>` 명령어로 테스트를 실행하여 정상 작동을 확인합니다.
- **디버깅**: 테스트 오류가 발생할 시 즉시 디버깅을 수행하여 문제를 해결합니다.
